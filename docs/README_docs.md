# Документация проекта NexusCRM

## Файлы

| Файл | Назначение |
|---|---|
| `NexusCRM_Documentation.docx` | Редактируемый исходник документа (Microsoft Word) |
| `NexusCRM_Documentation.pdf` | Готовый PDF для сдачи |
| `generate_doc.py` | Генератор `.docx` (python-docx) |
| `make_pdf.py` | Генератор `.pdf` (ReportLab + matplotlib) |

---

## Как пересобрать PDF

### Требования

```
pip install python-docx reportlab matplotlib pillow docx2pdf
```

### Генерация DOCX

```bash
py generate_doc.py
```

Создаёт `NexusCRM_Documentation.docx`.

### Генерация PDF

```bash
py make_pdf.py
```

Создаёт `NexusCRM_Documentation.pdf` через ReportLab с Times New Roman.

## Что не удалось автоматизировать

- **Автоматическое содержание с точными номерами страниц** — в `.docx` номера страниц в оглавлении указаны приблизительно (таблица TOC без полей Word). Рекомендуется обновить содержание в Word: `Ctrl+A → F9`.
- **Конвертация docx→pdf через Word** — требует Microsoft Word с правами COM-автоматизации; в данной среде не доступна. PDF сгенерирован через ReportLab напрямую.
- **Кавычки «ёлочки»** — в ReportLab/HTML-контексте используются Unicode-символы « », что эквивалентно требованию методички.
